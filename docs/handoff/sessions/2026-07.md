Resume per adr-0006's combined model: `apply --resume-run-id/--resume-manifest` (repeatable — a multiply-interrupted run passes every prior manifest) builds an action_id→record map from prior `result=applied` records and reconciles before the per-action ladder: live output matches `after_sha256` → `already-applied` skip; changed/missing → failed ERR-002 (external interference surfaces, never silently proceeds); unrecorded → normal execution (FR-003 guard covers retry-after-failure). Design decisions recorded here: (a) resume writes its own manifest (new run-id) rather than appending to the prior one — run identity, sidecar naming, and the lock stay simple; "manifest identical to uninterrupted run" is satisfied as union-of-records covering each action exactly once (asserted in the e2e); restoring a resumed run replays each manifest LIFO, newest first (runbook).
