# Architecture

## Component Graph

docmend is a single-process, layered pipeline (spec §8.1): Discovery -> Planning -> Transform -> Writer -> Verification, each stage living under `src/docmend/` once built. Only the Writer layer touches the filesystem for mutation; Transform is pure functions (text in, text out). Full diagrams: spec §8.2.

## Current Shape

Through MS-3 (2026-07-07, PR #9). New since MS-2 — the write path: `writer/atomic.py` (temp+fsync+`os.replace`, link-based no-clobber rename/publish with rollback, `rename_overwrite` — the NFR-002 primitives; the only os-level mutation calls in the codebase), `writer/backup.py` (verify-then-mutate tool backups, `<backup_root>/<run_id>/<rel>`, FR-006/ERR-004), `writer/gate.py` (adr-0004 pure-predicate safety gate + `ApplyOptions`; risk tiers per OQ-035), `writer/manifest.py` (DR-004 `ManifestRecord`/`ManifestWriter` — lazy-open, fsync per record + parent-dir fsync on create — and the adr-0006 AOF reader), `writer/apply.py` (the engine: hash guard -> snapshot filters -> resolve-containment -> recompute/cross-check -> EC-005 -> collision policy -> dry-run boundary -> backups -> atomic mutation -> manifest record), `report.py` (DR-003 models), `restore.py` (IR-008 LIFO preflight-then-mutate replay, mode-preserving), `lock.py` (flock(2) run lock keyed on hashed resolved source root under `$XDG_STATE_HOME/docmend/locks/`, OQ-036), and `cli.py` gained `apply`/`restore`. Schemas at inventory 1.1 / plan 1.1 (optional `source_root`) / manifest 1.1 / report 1.0. Command `verify` does not exist yet (MS-4).

Through MS-2 (2026-07-06). `src/docmend/`: `cli.py` (typer shell: IR-005 global flags + the `scan`/`plan` commands; artifacts/log default into `./.docmend/`, OQ-034 assumption), `config.py` (§18.2 strict pydantic models), `observability.py` (structlog via stdlib handlers; run-ID convention), `discovery.py` (read-only walk + single-pass chunked classifier; pathspec `GitIgnoreSpecPattern` filters; calls `detection.py` for the legacy encoding rung), `detection.py` (the `charset-normalizer`-sole legacy detection rung, adr-0009 — policy-free, returns `DetectedEncoding | None`), `inventory.py` (DR-001 internal models), `plan.py` (DR-002 plan/action/skip models + the `changed-since-scan` pre-implementation v1.0 schema amendment), `planning.py` (the domain layer: fact-gate ladder -> content pass -> plan assembly, FR-002/FR-015/DR-002; the sole module that reads library bytes read-only pre-apply), `transform/` (`encoding.py` strict BOM-aware decode/UTF-8-no-BOM encode; `newlines.py` LF normalization; `whitespace.py` the trim/final-newline/blank-line-collapse/leading-tab quartet; `dispatch.py` the adr-0016 file-class dispatch seam + EC-005 non-whitespace-count invariant metric — all NFR-005-pure, no filesystem access), `artifacts.py` (schema registry + cached Draft 2020-12 validators with format assertion + atomic artifact IO, extended with plan read/write), `schemas/` (the four hand-authored artifact JSON Schemas — the adr-0005 durable contract, shipped inside the wheel), and an empty `writer/` package anchoring the NFR-005 purity contract for MS-3. `tests/corpus.py` holds the shared seeded recipe->bytes generator (promoted out of `tests/test_discovery.py` per adr-0015). Commands `apply`/`verify`/`restore` do not exist yet.

## Standing Backlog

- Milestone order is binding (spec §19, Appendix B.1): MS-0 Foundation -> MS-1 Core workflow -> MS-2 Domain logic -> MS-3 CLI experience -> MS-4 Unattended operation -> MS-5 Production readiness. Do not build a later milestone on an unproven earlier one.
- Open decisions: OQ-034 (`.docmend/` artifact convention), OQ-035 (preservation flags + risk tiers), OQ-036 (flock run lock + the documented apply-vs-restore lock-key gap) — all non-blocking, owner sign-off wanted by MS-4; DEV-001 (EC-011 plan-time hard-link skip) pending owner review. Everything else is settled (`docs/resolved-questions.md`, ADRs 0001–0017). The MS-2 encoding-floor calibration (RQ-022) is resolved — floor stays 20 (adr-0009 note); the residual cp1252-vs-cp1250 case is pinned as risk marker R-001.
- MS-4 is unattended operation: resume model (FR-013 — adr-0006 reconciliation over the landed seq/fsync/AOF manifest), `verify` (FR-014), idempotency (FR-017), stale-plan tests. Recorded MS-4 inputs: manifest 1.2 `source_root` field + restore lock rekey (OQ-036 gap); the deferred MS-3 final-review minors (`sessions/2026-07.md`).
- Deferred capabilities (spec §2.3, WH-001-WH-008): semantic renaming, spelling/grammar repair, document reconstruction, HTML structural conversion, deduplication, frontmatter enrichment, search integration, and the low-ceremony one-shot command — none are v1 scope.
