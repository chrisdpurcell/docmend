- 2026-07-07 — MS-4 items 3–5: resume (FR-013), idempotency (FR-017), NFR-006 verify leg — **MS-4 complete** (spec rev 0.18). Resume per adr-0006's combined model: `apply --resume-run-id/--resume-manifest` (repeatable — a multiply-interrupted run passes every prior manifest) builds an action_id→record map from prior `result=applied` records and reconciles before the per-action ladder: live output matches `after_sha256` → `already-applied` skip; changed/missing → failed ERR-002 (external interference surfaces, never silently proceeds); unrecorded → normal execution (FR-003 guard covers retry-after-failure). Design decisions recorded here: (a) resume writes its own manifest (new run-id) rather than appending to the prior one — run identity, sidecar naming, and the lock stay simple; "manifest identical to uninterrupted run" is satisfied as union-of-records covering each action exactly once (asserted in the e2e); restoring a resumed run replays each manifest LIFO, newest first (runbook). (b) `already-applied` is excluded from the exit-1 finding count — otherwise a clean resume exits 1 and unattended re-invocation loops never converge; it's a new internal `ApplySkipReason` only (report schema `skip_reason` is free-string, no bump). (c) A wrong-tree resume manifest refuses exit 2 by cross-checking the manifest's stamped 1.2 `source_root` against the plan's — action-IDs could never match, and mass re-execution would mask the operator's mix-up. (d) The lost-trailing-record crash shape (mutation durable, record torn) resumes as a stale-hash/unreadable SKIP, not corruption — the tool can't prove that file's state, so it stays a reviewable finding; pinned in `test_resume_after_lost_trailing_record__stale_hash_skip_not_corruption`. Interruption injected as a RuntimeError subclass through `atomic_write_bytes` (bypasses the per-action `except (WriteError, OSError)` exactly like a kill). FR-017 closed with the three duplicate shapes (`tests/test_idempotency.py`); blind double-apply exits 1 by design (without --resume, "already converted" and "changed under me" are indistinguishable — AW-004 posture).
