# Architecture

## Component Graph

docmend is a single-process, layered pipeline (spec ¬ß8.1): Discovery -> Planning -> Transform -> Writer -> Verification, each stage living under `src/docmend/` once built. Only the Writer layer touches the filesystem for mutation; Transform is pure functions (text in, text out). Full diagrams: spec ¬ß8.2.

## Current Shape

**DMR-08 sequential million-file scale contract (closed 2026-07-18).** The v2.0.0 path deliberately remains sequential: a deterministic streamed corpus feeds fresh installed-wheel scan, plan, apply, and verify supervisors; strict aggregate-only evidence binds exact clean-HEAD provenance, reference identity, corpus/finding conservation, artifact validity, external peak RSS, child swap, linearity, capacity, and runtime. Plan schema 2.0 and configuration no longer expose inert parallel settings. The default gate carries a 1,000-file source-tree guard, while reviewed installed-wheel evidence owns the 100,000-file pilot, 100 MiB file-size matrix, and binding one-million-file release proof. Stage supervision samples a recoverable post-sample `/proc/<pid>/status` gap before polling because `Popen.poll()` may reap the terminal zombie snapshot; invalid or unreadable post-sample telemetry at exit fails closed ([BUG-001](bugs/001-scale-swap-telemetry-reap-race.md)). Accepted release candidate `ae3a286` completed in 25,629.225 seconds with zero child swap and a 20,825,497,600-byte maximum stage RSS.

**DMR-08 Task 11 file-size qualification lane (2026-07-15, `48d148c`).** The repo-only installed-wheel orchestrator now supports a binding `file-size` tier derived from `limits.max_file_size_mib`: one control plus upward-rounded quartiles for UTF-8 and Windows-1252, with external preservation throughout and tool backups on the bounded maximum-sized subset. Every case uses fresh scan/plan/apply/verify supervisors and records aggregate-only RSS, swap, elapsed, watchdog, backup, artifact, plan-coverage, and finding evidence. Scale-evidence 3.0's previously reserved `file-size` tier now has a strict tier-specific payload while existing cardinality evidence remains valid. Binding acceptance still requires an exact clean-HEAD wheel, reference match, all cases below the strict 2 GiB RSS envelope, zero child swap, complete reconciliation, and no-clobber publication.

The temporary root `ALIGNMENT_HANDOFF.md` was deleted once adr-0018 and its sibling were accepted. Schemas: inventory 1.2 / plan 1.2 / report 1.0 / **manifest 1.3** / frontmatter 1.0.
